## 一、市場調查摘要

### 1.1 市場規模與趨勢

全球記帳追蹤 App 市場預計從 2025 年的 108.6 億美元成長至 2026 年的 122.6 億美元（CAGR 12.8%）。亞太地區以 17.1% 的最高複合增長率領先，受益於電子發票法規與行動裝置的普及。

**關鍵趨勢**: AI 驅動洞察、語音辨識助手、訂閱管理、協作式預算規劃正在成為主流功能。

### 1.2 競品分析

| 競品 | 語音功能 | AI 能力 | 收費模式 | 優勢 | 痛點/缺口 |
|------|---------|---------|---------|------|-----------|
| **哈啦Money** (台灣) | 基礎語音輸入 | 規則式分類 | 免費+廣告 | 台灣在地化、發票掃描 | 分類不智慧、UI老舊、廣告多 |
| **TalkieMoney** (台灣) | AI語音 | 自動分類+智能查詢 | Freemium (50筆免費) | AI分類準確、對話式查詢 | 免費額度低、無發票整合 |
| **MonAi** | 自然語言 | AI 洞察 | 訂閱制 | Apple Pay整合、多幣別 | 僅iOS優化、中文支援弱 |
| **ExpenseEasy** | Whisper語音 | 自動提取 | Freemium | 多語言、免手動 | 功能單一 |
| **Say** | 語音+SMS | AI洞察 | Freemium | 銀行簡訊自動匯入 | 區域限制、隱私疑慮 |

### 1.3 市場痛點（用戶需求缺口）

1. **語音辨識不夠聰明** — 現有App僅做語音轉文字加規則比對，無法理解上下文語境
2. **缺乏主動式理財洞察** — 用戶記了帳卻不知道怎麼優化支出
3. **多筆交易一次說完做不到** — 現有App一次只能記一筆
4. **台灣在地化+國際化平衡差** — 要嘛只做台灣要嘛不做台灣
5. **無法與生活情境深度整合** — 沒有結合行事曆、位置、習慣的智能提醒

## 二、AI 獨特優勢與差異化特色

### 2.1 核心 AI 差異化（競品做不到的事）

#### 特色一：語境感知多筆記帳 (Context-Aware Batch Entry)
> **使用者說**: 「今天中午跟同事去吃鼎泰豐花了 520，下午在全聯買了牛奶麵包 187，回家路上加油 1200」
>
> **AI 做到**: 一次語音輸入 → 自動拆分三筆 → 分類（餐飲/日用品/交通）→ 標記時間地點 → 辨識「跟同事」為社交場景

技術關鍵: LLM 語意解析 + NER（命名實體識別）+ 時序推理

#### 特色二：AI 財務教練 (Proactive Financial Coach)
> 不只記帳，主動分析消費模式並給出可執行建議
>
> - 「你本月外食已超過預算 73%，還有 8 天，建議這週自己煮」
> - 「你每週五固定消費較高，可能是聚餐日，要設定限額嗎？」
> - 「你的通勤支出比上月多 20%，是否換了交通方式？」

技術關鍵: 時序分析 + 行為模式辨識 + 個人化推薦引擎

#### 特色三：對話式帳本查詢 (Conversational Ledger)
> **使用者問**: 「上個月我在飲料上花了多少？比前一個月多嗎？」
>
> **AI 回答**: 「上月飲料消費 1,280 元（23 筆），比前月多了 340 元（+36%），主要增加在手搖飲，平日下午 2-4 點最常買」

技術關鍵: Text-to-SQL + 自然語言生成 + 統計摘要

#### 特色四：智能發票 OCR + 語音補充 (Hybrid Receipt Intelligence)
> 拍照掃描發票/收據 → AI 自動辨識品項金額 → 語音補充備註
> 「這張是跟客戶的商務餐費，列入報帳」

技術關鍵: Vision Model OCR + 語音指令融合

#### 特色五：消費預測與異常偵測 (Spending Prediction & Anomaly Detection)
> - 根據歷史資料預測月底支出
> - 偵測異常消費（突然多了一筆大額支出自動提醒確認）
> - 月初預算建議基於上月實際數據

技術關鍵: 時序預測模型 + 異常偵測算法

## 三、技術架構與開發框架

### 3.1 框架選擇：Flutter（推薦）

選擇理由:
- 效能優勢: Dart 編譯為原生代碼，無 JS Bridge 開銷，語音處理延遲更低
- 跨平台一致性: iOS/Android/Web 一套程式碼，UI 完全一致
- 語音整合成熟: speech_to_text、flutter_sound 等套件生態完善
- 2026 趨勢: Flutter 在動畫與自訂 UI 上持續領先
- 單人/小團隊友善: 一套代碼維護成本低

### 3.2 技術棧

前端: Flutter 3.x + Dart 3.x + Riverpod + FL Chart + Google ML Kit
後端: Supabase (PostgreSQL + Auth + Storage) + Edge Functions (Deno)
AI: OpenAI Whisper API + Claude/GPT API
自動化: n8n + GitHub Actions
基礎設施: Firebase (推播) + RevenueCat (訂閱管理)

### 3.3 n8n 自動化工作流程設計

1. CI/CD Pipeline: GitHub Push → n8n → Flutter Test → Dart Analyze → Build → Deploy
2. AI 模型更新: 回饋收集 → 分類準確度統計 → 模型微調 → A/B 測試
3. 用戶洞察通知: Cron → Supabase 查詢 → AI 消費摘要 → FCM 推送
4. Issue 自動管理: App 反饋 → AI 分類 → GitHub Issue → 標籤 → 通知

## 五、收費模式設計 (Freemium 訂閱制)

### 免費版 (Free Tier)
- 基礎語音記帳（每月 30 筆）、手動輸入不限、基礎分類（10 類）、月度摘要、基礎圖表

### 進階版 Pro (NT$ 99/月 或 NT$ 899/年)
- 無限語音記帳（核心收費點）、AI 多筆拆分、自訂類別、AI 財務教練、發票 OCR、匯出報表、多幣別

### 旗艦版 Premium (NT$ 199/月 或 NT$ 1,799/年)
- Pro 全部 + 對話式帳本查詢（高級收費點）、消費預測、家庭共享（5人）、API 整合、專屬 AI 客服
